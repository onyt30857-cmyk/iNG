## 5. 复盘场景的 7 个状态界面

复盘是产品最重要的场景。每个状态有不同的视觉处理。

### 5.1 ENTRY 状态

底部抽屉式弹出,占 60% 屏。

```
┌────────────────────────────┐
│              ━━━━           │ ← BottomSheet handle
│                            │
│  先把聊天截图给我看看       │
│                            │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐      │
│  │📷│ │  │ │  │ │+│      │
│  └──┘ └──┘ └──┘ └──┘      │
│                            │
│  最多 5 张,按时间顺序      │
│                            │
│  ┌──────────────────────┐  │
│  │  开始让老白看一下   │  │
│  └──────────────────────┘  │
└────────────────────────────┘
```

### 5.2 PARSING 状态

```
┌────────────────────────────┐
│ ←  老白在看                │
│                            │
│ [截图1] [截图2] [截图3]     │
│                            │
│ ┌──────────────────────┐   │
│ │ K                    │   │
│ │ 我看了下你和小雨这段。│   │
│ │ 从 4 月 12 号到昨晚, │   │
│ │ 大概一周时间,你发了  │   │
│ │ 14 条,她回了 8 条。  │   │
│ │                      │   │
│ │ 几个事:              │   │
│ │ - 她最后一条「先这样吧」│ │
│ │   是昨晚 9 点 47..._  │   │← 流式输出 + 闪烁光标
│ └──────────────────────┘   │
│                            │
└────────────────────────────┘
```

视觉重点:
- 顶部:截图缩略图横向排列(可点击放大)
- 中部:老白的话以打字机效果一行一行出现(30ms/字符)
- 底部:暂时空白,等 PARSING 结束才显示"继续"按钮

### 5.3 REFLECTING 状态

```
┌────────────────────────────┐
│ ←  和老白聊聊              │
│                            │
│ K 看完这段你最在意她哪一句?│
│   为什么是那一句?           │
│                            │
│ ┌────────────────────────┐ │
│ │ 想到啥写啥,写错了也行  │ │
│ │                        │ │
│ │ ...                    │ │
│ │                        │ │
│ └────────────────────────┘ │
│                            │
│              [发送 ▶]      │
└────────────────────────────┘
```

关键:
- 老白的话以对话气泡形式呈现
- 三个问题不是同时出现,**一个一个出来**(等用户答完才出下一个)
- 用户输入框常驻底部
- 输入框上方小提示:"想到啥写啥,写错了也行"

### 5.4 DIAGNOSING 状态

**这是产品价值最高的瞬间**。视觉要让这个瞬间安静、隆重。

视觉:
- 老白的话以散文式段落呈现,不是点列
- 字号比平时大半号(17pt 而非 16pt)
- 段落间距比平时大(20px 而非 16px)
- 如果触发羞耻处理,这一段用淡紫色背景的卡片包裹

```scss
.diagnosing-message {
  font-size: 17pt;
  line-height: 1.7;
  
  p {
    margin-bottom: 20px;
  }
  
  .shame-handling-block {
    background: var(--color-primary-subtle);
    border-radius: var(--radius-md);
    padding: 16px 20px;
    margin: 24px 0;
    border-left: 3px solid var(--color-primary);
  }
}
```


## 8. 多线全景视图

视觉规格:
- 每个关系是一个圆角卡片
- 卡片内只显示:名字、关系阶段、上次复盘距今多久
- **绝不**做谁更喜欢你、谁回复率高、谁兴趣度强
- 整体感觉像"关系仪表板"而非"比赛排行榜"
